# Backbone_Perturbation
Adding perturbations to the cyclic peptide backbones generated by AI models such as BoltzGen.
To show how to use this, an example run is conducted.
The input peptides are stored in folder top100fullStructure.
To correctly read in the peptides, the .cif files need to be converted to .pdb (using PyMol or other tools).
The main function is implemented in backbone_perturbation.m, and all auxiliary functions are implemented in the folder my_functions.
The output perturbed peptide backbones will be stored in the folder perturbed_backbones.

To run backbone_perturbation.m, it requires the pdb filename of the input peptide. It first check if the peptide backbone has a good bond geometry, by looking at how much the bond lengths, bond angles, and omega torsion angles deviate from the ideal values. If the deviations are within the given thresholds, it will proceed to backbone perturbation; otherwise, the code will generate a warning and stop. To add perturbations, the code extracts the original phi, psi torsion angles and add small perturbations while satisfying the symmetric Glycine Ramachandran space. Besides maintaining a cyclic backbone, the code also checks for enough hydrogen bonds, no steric clashes inside the peptide and between the peptide and the target protein, and suitable distances between the peptide and the target binding site.
